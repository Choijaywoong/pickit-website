> 이 파일은 PICKIT 프론트엔드 구현의 UI 기준 문서다.
> 기능 명세는 `PRD.md`, 아키텍처 규칙은 `CLAUDE.md`를 우선 참조.
> 디자인 결정 충돌 시 이 파일이 기준이다.

## 0. 전체 레이아웃 원칙

### 구조
- **데스크탑**: 좌측 채팅 영역(주인공) + 우측 결과 패널(보조) — 2단 분할
- **모바일**: 채팅 풀스크린, 결과 패널은 하단 슬라이드업 시트로 전환
- 우측 패널은 **평소엔 숨겨진 상태(hidden)** — 주문 표·엑셀 미리보기·송장 입력 등이 필요한 응답 시에만 슬라이드로 열림
- React state `showPanel: boolean`으로 조건부 렌더링
- **우측 패널 최상단 오른쪽에 반드시 ✕ 닫기 버튼 포함** → 클릭 시 패널 숨김 + ChatWidget 전체 너비 복귀

### 레이아웃 비율
```
데스크탑: [채팅 flex:1] [패널 280px 고정]
모바일: [채팅 100vw] → 패널 필요 시 [바텀시트 60vh]
```

### PICKIT 응답 텍스트 규칙 (전체 화면 공통 — 반드시 준수)
- PICKIT의 모든 응답은 **배경 박스(말풍선) 없이 바탕에 텍스트만** 렌더링
- Claude UI처럼 흰 바탕에 텍스트가 그냥 써진 형태
- 사용자 입력만 파란 배경 말풍선 (우측 정렬)
- PICKIT 응답은 좌측 정렬 텍스트, 배경색 없음
- 응답 아래 빠른 액션 버튼은 작은 outline 버튼으로 나열 (배경 없음)

### 색상·상태 시스템 (전체 화면 공통)
| 색상 | 의미 | 사용 예 |
|------|------|---------|
| 초록 (success) | 정상·완료 | 채널 연동 정상, 송장 전송 완료 |
| 빨강 (danger) | 에러·즉시 조치 필요 | 연동 만료, 방화벽 트리거 |
| 노랑 (warning) | 경고·사전 알림 | 무신사 API 만료 임박, 에이블리 주의 |
| 파랑 (info) | 사용자 말풍선·정보 | 셀러 입력 채팅, 안내 강조 |
| 회색 (secondary) | 비활성·보조 | 미선택 채널, 비활성 UI |

**연결 FR**: FR-001
**파일 위치(예상)**: `src/components/Onboarding/StepInventory.jsx`

### 레이아웃

- 질문 텍스트 크게, 설명 텍스트 작게
- 선택지 카드 2개: 가로 나열 (모바일은 세로 스택)

### 컴포넌트 명세

**선택 카드 — "네, 보유합니다"**
- 파란 테두리 강조 (`border: 2px solid info`)
- 아이콘: `ti-package`
- 부제: "사입·자체브랜드 셀러. 발주 예측 알림이 켜집니다."
- 클릭 시: `hasInventory = true` 저장 → 자동으로 Step B로 이동

**선택 카드 — "아니요"**
- 기본 테두리 (강조 없음)
- 아이콘: `ti-truck-delivery`
- 부제: "위탁·드롭쉬핑. 재고 도미노 기능 추후 지원 예정."
- 클릭 시: `hasInventory = false` 저장 → 자동으로 Step B로 이동

### Acceptance Criteria (UI 기준)
- [ ] 두 카드 중 하나를 클릭하면 자동으로 Step B로 넘어간다 (별도 '다음' 버튼 불필요)
- [ ] 선택된 카드는 파란 테두리로 강조된다

**연결 FR**: FR-001
**파일 위치(예상)**: `src/components/Onboarding/StepChannels.jsx`

### 레이아웃

### 채널 카드 상태
| 상태 | 스타일 |
|------|--------|
| 미선택 | 기본 배경, 회색 테두리 0.5px |

```
아이콘: ti-alert-triangle (노란색)
제목: "에이블리는 '셀러스(Sellers)' 입점 타입만 연동 가능합니다"
본문: "입점 타입을 먼저 확인해 주세요. 다른 타입은 연동되지 않습니다."
배경: warning 배경, warning 테두리
```

### Acceptance Criteria (UI 기준)
- [ ] 에이블리 선택 즉시 경고 박스가 렌더링된다
- [ ] 2개 미만 선택 시 "시작하기" 버튼이 비활성(disabled)이다
- [ ] 선택 채널 목록이 전역 state에 저장된다

## 3. 첫 진입 화면 — 연동 상태 체크

**연결 FR**: FR-008
**파일 위치(예상)**: `src/components/ConnectionStatus.jsx`
**노출 시점**: 온보딩 완료 직후. 이후 재방문 시 채팅 화면 위에 배너 형태로도 활용 가능

### 레이아웃
- 상단: 타이틀 + 부제 ("N개 채널 중 M개 정상") + 새로고침 버튼
- 채널 카드 그리드: `repeat(auto-fit, minmax(200px, 1fr))`
- 하단: "채팅 시작하기" CTA 버튼 (전체 너비)

### 채널 카드 3종

**정상 카드**
```
배경: 기본 / 테두리: 기본 0.5px
왼쪽: 초록 점(8px) + 채널명
오른쪽: "정상" 텍스트 (초록)
```

**에러 카드 (연동 만료)**
```
배경: danger / 테두리: danger 0.5px
왼쪽: 빨간 점 + 채널명(빨간)
오른쪽: "만료" 텍스트 (빨간)
하단: 에러 설명 문구 + "다시 연결하기" 버튼 (전체 너비, 빨간 테두리)
```

**경고 카드 (만료 임박 — 무신사 전용)**
```
배경: warning / 테두리: warning 0.5px
왼쪽: 노란 점 + 채널명(노란)
오른쪽: "D-N" 형태 (예: D-23)
하단: "API 키 N일 후 만료. 미리 갱신하세요." 문구
```

### Acceptance Criteria (UI 기준)
- [ ] 카드별 상태 색상이 색상 시스템과 일치한다
- [ ] 에러 채널의 "다시 연결하기" 버튼 클릭 시 연동 설정 페이지로 이동한다
- [ ] 무신사 D-30 이내 시 경고 카드로 자동 전환된다

## 4. 메인 채팅 화면 — 주문 조회

**연결 FR**: FR-002, FR-003, FR-004
**파일 위치(예상)**: `src/components/ChatWidget.jsx`, `src/components/ResultPanel.jsx`

### ChatWidget 구조
```
┌─ 헤더 (고정) ──────────────────────────────────┐
│  PICKIT  [N채널 연동 중 뱃지]                  │
├─ 메시지 목록 (scroll) ─────────────────────────┤
│  사용자 말풍선 (우측, 파란 배경)               │
│  PICKIT 응답 (좌측, 배경 없는 텍스트)          │
│    └ 빠른 액션 버튼 (outline, 소형)            │
├─ 하단 엑셀 바 (주문/엑셀 응답 시에만 표시) ───┤
│  [스프레드시트 아이콘] [파일명] [건수]         │
│                          [펼치기] [저장]        │
├─ 입력창 (고정) ────────────────────────────────┤
│  [텍스트 입력 flex:1]  [전송 버튼]             │
└────────────────────────────────────────────────┘
```

### 말풍선 스타일
| 종류 | 정렬 | 스타일 |
|------|------|--------|
| 사용자 입력 | 우측 | 파란 배경, `border-radius: 12px 12px 2px 12px` |
| PICKIT 응답 | 좌측 | **배경 없음**, 텍스트만, `color: primary` |

### 하단 엑셀 바
- **노출 조건**: 주문 조회 응답 또는 엑셀 추출 응답이 있을 때만 표시
- 구조: `[ti-file-spreadsheet 아이콘(초록)] [파일명] [건수] | [펼치기 버튼] [저장 버튼]`
- "펼치기" 클릭 → ResultPanel 열림 + 체크박스 필터 활성화
- "저장" 클릭 → 전체 데이터 xlsx 즉시 다운로드 (클라이언트 사이드)
- 파일명 포맷: `YYYY_MM_채널명_매출_리포트.xlsx` (멀티채널이면 `전체채널`)

### ResultPanel 구조
```
┌─ 패널 헤더 ──────────────────────────────────┐
│  [제목 · N건]         [다운로드] [✕ 닫기]   │
├─ 체크박스 필터 (연동 채널이 2개 이상일 때만) ┤
│  플랫폼 선택                                  │
│  ☑ 쿠팡      83건                            │
│  ☑ 네이버    71건                            │
│  ☐ 카페24    28건                            │
│  ☐ 에이블리  22건                            │
│  ☐ 지그재그  10건                            │
│  선택: N건   [선택 항목 저장]                 │
├─ 데이터 표 (scroll) ─────────────────────────┤
│  채널 | 상품명 | 옵션 | 수량 (주문 조회)     │
│  채널 | 상품명 | 금액 (엑셀 미리보기)        │
└──────────────────────────────────────────────┘
```

### 체크박스 필터 규칙
- **연동 채널 1개**: 체크박스 영역 미표시 (필터 의미 없음)
- **연동 채널 2개 이상**: 체크박스 영역 항상 표시
- 기본값: 전체 채널 체크된 상태
- 복수 선택 가능 (중복 체크 허용)
- 체크 변경 시 아래 표와 "선택: N건" 카운트 실시간 업데이트
- "선택 항목 저장" → 체크된 채널 데이터만 필터링해서 xlsx 생성
- 채널명 색상: 쿠팡(파랑/info), 네이버(초록/success), 나머지는 기본색


### Acceptance Criteria (UI 기준)
- [ ] PICKIT 응답에 배경 박스가 없고 텍스트만 렌더링된다
- [ ] 주문 조회 응답 시 하단 엑셀 바가 나타난다
- [ ] 우측 패널 최상단 오른쪽에 ✕ 버튼이 있고 클릭 시 패널이 닫힌다
- [ ] 채널 2개 이상일 때만 체크박스 필터가 표시된다
- [ ] 체크박스 변경 시 표 데이터와 건수가 실시간으로 필터링된다
- [ ] "선택 항목 저장"은 체크된 채널 데이터만 xlsx로 생성한다

## 4-1. 송장 입력 화면 — 인라인 패널

**연결 FR**: FR-004
**파일 위치(예상)**: `src/components/InvoicePanel.jsx`
**렌더링 방식**: 채팅 트리거 → 우측 ResultPanel 영역에 슬라이드로 열림

### 진입 방식
- 사용자가 채팅에 "송장 입력할게", "오늘 송장 넣어줘", "송장 전송해줘" 등 입력 시
- ResultPanel이 열리며 송장 입력 UI 렌더링
- 채팅 PICKIT 응답: "오늘 미처리 주문 N건입니다. 우측에서 송장 번호를 입력해 주세요."

### 레이아웃
```
┌─ 패널 헤더 ──────────────────────────────────┐
│  송장 입력 · 미처리 N건   [전체 전송] [✕]   │
├─ 풀필먼트 안내 (해당 주문 있을 경우만) ──────┤
│  ⓘ N건은 플랫폼 배송 위탁 건 — 입력 불필요  │
├─ 주문 목록 (scroll) ─────────────────────────┤
│  [채널뱃지] 상품명 (옵션)           수량     │
│  택배사 선택 ▾  [송장번호 입력________]  ✓  │
│  ─────────────────────────────────────────── │
│  (반복)                                       │
├─ 하단 고정 ───────────────────────────────────┤
│  입력 완료 N건 / 전체 N건  [선택 전송] [전체 전송] │
└──────────────────────────────────────────────┘
```

### 주문 행 컴포넌트 명세
| 요소 | 상세 |
|------|------|
| 채널 뱃지 | 쿠팡(파랑), 네이버(초록), 기타 기본색 — 작은 pill 형태 |
| 상품명 + 옵션 | 예: "블랙 린넨 셔츠 (L / 블랙)" |
| 수량 | 우측 정렬, 회색 |
| 택배사 선택 | 드롭다운: CJ대한통운 / 한진 / 로젠 / 우체국 / 기타 |
| 송장번호 입력창 | `placeholder="송장번호 13자리"`, 숫자만 입력 허용 |
| 완료 표시 ✓ | 송장번호 입력 완료 시 행 배경 연한 초록 + 우측 ✓ 아이콘 |
| 체크박스 | 행 좌측에 위치 — "선택 전송" 대상 지정용 |

### 풀필먼트 주문 처리
- 배송 타입이 "플랫폼 위탁(로켓배송, NFA 등)"인 주문은 목록에서 회색 처리
- 해당 행의 택배사 선택·송장번호 입력창 비활성(disabled)
- 행 우측에 "플랫폼 배송" 회색 뱃지 표시
- 패널 상단 안내 문구는 풀필먼트 주문이 없으면 미표시

### 전송 버튼 규칙
| 버튼 | 조건 | 동작 |
|------|------|------|
| 선택 전송 | 체크박스로 선택한 행만 | 선택 건만 각 채널 API로 전송 |
| 전체 전송 | 송장번호 입력 완료된 전체 행 | 입력 완료 건 전체 일괄 전송 |
| 헤더 전체 전송 | 상동 | 동일 동작 (헤더 접근성용 중복 배치) |

### 전송 후 상태 처리
- **성공한 행**: 초록 배경 유지 + "전송 완료" 텍스트로 변경
- **실패한 행**: 빨간 배경 + "전송 실패 — 재시도" 버튼 노출
- **전체 완료 시**: 채팅 영역에 채널별 결과 요약 텍스트 자동 출력
  - 예: "쿠팡 12건, 네이버 8건 송장 전송 완료. 카페24 2건 실패 — 재시도하시겠어요?"

### Acceptance Criteria (UI 기준)
- [ ] 채팅 송장 트리거 입력 시 ResultPanel이 열리며 미처리 주문 목록이 렌더링된다
- [ ] 풀필먼트 주문은 입력창이 비활성화되고 "플랫폼 배송" 뱃지가 표시된다
- [ ] 풀필먼트 주문이 없으면 상단 안내 문구가 표시되지 않는다
- [ ] 송장번호 입력 완료 시 해당 행 배경이 연한 초록으로 변한다
- [ ] 전송 성공/실패가 행 단위로 구분되어 표시된다
- [ ] 전체 완료 시 채팅 영역에 채널별 결과 요약이 출력된다
- [ ] 숫자 이외의 문자는 송장번호 입력창에서 입력되지 않는다

## 5. 방화벽 — 하단 알림창

**연결 FR**: FR-007
**파일 위치(예상)**: `src/components/FirewallToast.jsx`
**렌더링 위치**: ChatWidget 내 입력창 바로 위에 `position: absolute`로 오버레이

### 트리거 조건 (백엔드 `isDangerous: true` 반환 시)
| 방화벽 종류 | 트리거 | UI 처리 |
|------------|--------|---------|
| 금액 방화벽 | 기존가 대비 ±20% 초과 또는 0원 이하 | 하단 알림창 |
| 범위 방화벽 | 5개 이상 또는 전체·모두·싹다·일괄 키워드 | 하단 알림창 |
| 맥락 방화벽 | Confidence < 85% | 알림창 아닌 "되물음" 텍스트 응답 |
| 플랫폼 불일치 | 미지원 채널 요청 | 알림창 아닌 안내 텍스트 응답 |

### 알림창 구조 (금액·범위 방화벽)
```
┌─ 알림창 헤더 (danger 배경) ──────────────────┐
│  🛡 [방화벽 종류명] — 승인이 필요해요        │
├─ 3개 정보 (가로 3열) ────────────────────────┤
│  [대상 채널]   [영향 상품]   [변경 내용]     │
├─ 액션 버튼 (2열) ────────────────────────────┤
│  [취소]              [최종 승인하기]          │
└──────────────────────────────────────────────┘
```

### 표시 정보 — 3개만 (트리거 이유 제외)
| 항목 | 내용 예시 |
|------|---------|
| 대상 채널 | "전체 (5개)" 또는 "쿠팡, 네이버" |
| 영향 상품 | "247개" — danger 색상으로 강조 |
| 변경 내용 | "가격 −500원" |

### 핵심 UX 규칙
```
알림창 표시 시:
- 입력창(input): disabled = true, opacity: 0.45
- 전송 버튼: disabled = true, opacity: 0.45
- ResultPanel: 잠금 상태 (자물쇠 아이콘 + "승인 후 결과 표시")
→ "최종 승인하기" 또는 "취소" 클릭 시 위 3개 모두 원상복귀
```

### 스타일
- 알림창 전체: `border: 1.5px solid danger`, `background: primary`, 그림자 `box-shadow: 0 4px 20px rgba(0,0,0,0.12)`
- 헤더: `background: danger 배경`
- "최종 승인하기" 버튼: `border-color: danger`, `color: danger` (의도적 빨간색)
- "취소" 버튼: 기본 회색 스타일
- 위치: `position: absolute; bottom: 64px; left: 12px; right: 12px; z-index: 10`

### Acceptance Criteria (UI 기준)
- [ ] 알림창이 입력창 바로 위에 떠오른다 (채팅 메시지 목록 안이 아님)
- [ ] 알림창 표시 시 입력창과 전송 버튼이 disabled 처리된다
- [ ] 3개 정보만 표시된다 (대상 채널 / 영향 상품 / 변경 내용)
- [ ] "취소" 클릭 시 알림창이 사라지고 입력창이 활성화된다
- [ ] "최종 승인하기" 클릭 시 백엔드로 최종 실행 요청이 전송된다
